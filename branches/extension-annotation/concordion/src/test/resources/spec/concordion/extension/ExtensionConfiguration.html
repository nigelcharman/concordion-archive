Extension Configuration

Extensions can be added to Concordion using an annotation in the fixture class and/or using a system property.

Annotations

@Extension

Within a fixture class, fields and methods that are annotated with @org.concordion.api.extension.Extension will be added to Concordion as extensions.

@Extension Fields

Fields with this annotation must be public and must implement org.concordion.api.extension.ConcordionExtension.

Example - Extensions as Fields

Executing the following fixture:

import org.concordion.api.extension.Extension;
import org.concordion.api.extension.ConcordionExtension;
import test.concordion.extension.*;
 
public class ExampleFixture {
    
    @Extension
    public ConcordionExtension extension = new DummyExtension1();
 
    @Extension
    public DummyExtension2 extension2 = new DummyExtension2();
}

will install both extensions DummyExtension1, DummyExtension2.

Extensions will be loaded from the fixture class and any of its superclasses. A common pattern is to have the extensions defined in a "base fixture".

Example - Extensions as superclass fields

Executing the following fixture:

import org.concordion.api.extension.Extension;
import org.concordion.api.extension.ConcordionExtension;
import test.concordion.extension.*;
 
public class ExampleFixture extends BaseFixture {
    @Extension
    public ConcordionExtension extension = new DummyExtension1("ExampleExtension");
 
}

which has superclass 

import org.concordion.api.extension.Extension;
import org.concordion.api.extension.ConcordionExtension;
import test.concordion.extension.*;
 
public class BaseFixture {
    
    @Extension
    public DummyExtension2 extension2 = new DummyExtension2("SuperExtension");
}

will install both the extensions initialised with parameters SuperExtension, ExampleExtension.

@Extension Methods

Methods marked with this annotation must take no parameters, be public and return an object that implements org.concordion.api.ConcordionExtension.

@Extensions

Where the extensions do not require any state from the fixture, they can be defined statically on the fixture class with the @org.concordion.api.extension.Extensions annotation. This annotation is parameterised with a list of the extension, or extension factory, classes to be installed.

System Property

Alternatively, extensions can be specified by setting a system property. This can be useful if the extensions need to be configured independently from the fixtures.

Set the system property concordion.extensions to a comma-separated list containing:

  • the fully-qualified class name of extensions to be installed, and/or
  • the fully-qualified class name of extension factories that will create an extension.

If an extension is specified, it will be instantiated by Concordion.

All extensions and/or extension factories must be present on the classpath. Extensions must implement org.concordion.api.extension.ConcordionExtension. Extension factories must implement org.concordion.api.extension.ConcordionExtensionFactory.
