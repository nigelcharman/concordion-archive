Extensions Library

The optional concordion-extensions library contains pre-built extensions that take advantage of the Extensions API. It was developed and is maintained by Nigel Charman.

The full source code (Apache 2 license) is available, which may be useful as a starting point for developing your own extensions.

Pre-built extensions

  • Screenshot Extension — add screenshots to Concordion output to diagnose problems or improve the documentation
  • Logging Extension — add logging information as tooltips to the Concordion output
  • Timestamp Formatter Extension — change the footer of the Concordion output to show hours, minutes and seconds
  • Translator Extension — modify the text of exception messages
  • Embed Extension — embed HTML in the Concordion output

Installation

The extensions are packaged in the concordion-extensions-1.x.jar available from the concordion-extensions download page. You will need to ensure that both the concordion-extensions and Concordion 1.4.1 or later (and its dependencies) are on your classpath.

Installing from Maven Central

Alternatively, this project is available in Maven Central, with a group id of org.concordion and an artifact id of concordion-extensions.

Example project

The extensions example project demonstrates the Screenshot, Logging Tooltip and Translator extensions using Concordion and Selenium WebDriver for end-to-end browser testing.

This project can be downloaded from the concordion-extension downloads page (the concordion-extensions-demo zip file), or can be obtained from the source repository. It includes Gradle and Maven builds to download dependencies and build the project. See the README.html file in the download for instructions.
